# Website Overhaul for Ghost.org Application — 3 Week Build Sprint

## Project Overview

This project is a live, production-grade application for the **Senior Platform Engineer** role at Ghost.org.

Instead of submitting only a resume and cover letter, I’m building a custom Ghost theme and supporting platform systems that demonstrate the role in practice. The end result will be a fast, polished, well-documented site hosted on **mannyroy.com**, with a dedicated Ghost application experience that showcases both the final product and the engineering process behind it.

The goal is not just to prove frontend polish, but to demonstrate platform thinking across:

- theme architecture
- performance engineering
- AI-assisted product development
- documentation systems
- observability
- deployment discipline
- reliability and maintainability
- technical communication
- public engineering storytelling

This will be built in a **maximum 3-week intensive sprint**, using AI-assisted coding to accelerate execution while maintaining production-level quality.

### Core thesis

I am using AI as a force multiplier to rapidly design, build, refine, document, and launch a high-quality platform artifact under real delivery pressure.

That is part of the application.

### Primary deliverable

A live Ghost-powered site and custom theme that functions as:

- a professional application artifact
- a technical case study
- a documentation system
- a public demonstration of platform engineering judgment
- a proof point for high-velocity, AI-assisted execution

### Audience

This project serves two audiences:

1. **Ghost hiring team / leadership**
   - concise, polished, high-signal proof of fit

2. **Engineers / technical reviewers**
   - architecture, tradeoffs, implementation detail, operations, and documentation depth

### Success criteria

By launch, the project should clearly demonstrate:

- excellent visual and UX quality
- strong performance and Core Web Vitals
- clear system architecture
- disciplined use of AI in production-minded delivery
- thoughtful documentation of technical decisions
- operational maturity in deployment and monitoring
- strong alignment with Ghost’s platform engineering responsibilities

# Principles for AI-assisted development

This project intentionally uses AI to accelerate development.

The point is not to hide that. The point is to demonstrate mature use of it.

## Rules

- AI accelerates implementation, not judgment
- architecture and tradeoffs remain human-owned
- every major feature must still be testable, understandable, and maintainable
- speed is valuable only if quality remains legible
- public writeups should distinguish between assisted implementation and core engineering decisions

## What this proves

- rapid prototyping under pressure
- effective decomposition of work
- ability to use modern tooling pragmatically
- high output velocity without abandoning engineering discipline

# Risks

## Risk: overscoping

Too many features could weaken execution quality.

### Mitigation

- maintain a strict v1 / v2 cutline
- prioritize clarity, polish, and proof over novelty

## Risk: AI feature becomes gimmicky

A generic chatbot would reduce credibility.

### Mitigation

- make the assistant narrow, useful, grounded, and well-documented

## Risk: documentation lags behind implementation

This would weaken the “process” story.

### Mitigation

- publish documentation as features land, not only at the end

## Risk: polish suffers under time pressure

### Mitigation

- prioritize a professional core experience first
- only add advanced features after the core is excellent
