This docs site is organized for three core audiences. Instead of navigating by “setup steps”, each path leads with the story and the artifacts that matter most to that audience.

## Operator (Ops & SRE) — run & maintain

## Reader — what it does

- Start with: [Quick start — Reader](./quick-start/quick-start-reader)

## If you’re not sure

Pick Developer if you’re changing code or templates. Pick Operator if you’re deploying, monitoring, or troubleshooting. Pick Reader if you want the outcomes, privacy, and FAQ first.
