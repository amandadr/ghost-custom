# Changelog

**TL;DR:** When we ship a meaningful change, we record a short release note here using a consistent format (what changed, why it mattered, and what to check).

## Release note format

### Version: `X.Y.Z`

- Date: `YYYY-MM-DD`
- Theme (Ghost zip): 1–3 sentences
- Docs (Docusaurus): 1–3 sentences
- Operations / assistant: 1–3 sentences (health checks, runbook updates, widget changes)
